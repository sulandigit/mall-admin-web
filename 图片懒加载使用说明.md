# 图片懒加载功能使用说明

## 功能概述

我们为商城后台管理系统实现了图片懒加载功能，能够显著提升页面性能，特别是在商品列表等包含大量图片的页面中。

## 技术实现

### 1. 自定义指令 `v-lazy`

创建了一个Vue自定义指令，使用Intersection Observer API实现图片的按需加载。

**特性：**
- 支持占位符图片
- 支持加载失败图片
- 平滑的过渡动画
- 自动清理observer避免内存泄漏
- 支持动态更新图片地址

### 2. 使用方式

#### 基本用法
```html
<img v-lazy="imageUrl" alt="商品图片">
```

#### 高级用法（带配置）
```html
<img v-lazy="{
  src: imageUrl,
  placeholder: placeholderUrl,
  onLoad: handleLoad,
  onError: handleError
}" alt="商品图片">
```

### 3. 样式类

- `.lazy-image` - 基础懒加载图片样式
- `.lazy-loading` - 加载中状态
- `.lazy-loaded` - 加载完成状态
- `.lazy-error` - 加载失败状态
- `.product-image-lazy` - 商品图片专用样式

## 性能优化效果

1. **减少初始加载时间** - 只加载视口内的图片
2. **节省带宽** - 用户不滚动到的图片不会被加载
3. **提升用户体验** - 平滑的加载动画和占位符
4. **减少服务器压力** - 分批加载图片，避免并发请求过多

## 配置说明

### 观察器配置
- `rootMargin: '50px'` - 提前50px开始加载
- `threshold: 0.01` - 图片1%进入视口时触发加载

### 默认图片
- 使用SVG占位符，轻量且美观
- 错误图片使用红色X标记，清晰表示加载失败

## 扩展使用

可以在其他需要图片懒加载的组件中使用：

```html
<!-- 商品详情页图片 -->
<img v-lazy="product.albumPics[0]" class="product-detail-image">

<!-- 品牌Logo -->
<img v-lazy="brand.logo" class="brand-logo">

<!-- 用户头像 -->
<img v-lazy="user.avatar" class="user-avatar">
```

## 注意事项

1. 确保图片容器有明确的尺寸，避免布局抖动
2. 为图片添加有意义的alt属性，提升无障碍性
3. 对于关键图片（首屏），可以考虑预加载而不是懒加载
4. 在低网络环境下，可能需要调整rootMargin参数
