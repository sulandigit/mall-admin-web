# 性能监控Dashboard使用说明

## 概述

本性能监控dashboard为mall后台管理系统提供了全面的系统性能监控功能，包括系统资源、网络性能、数据库性能、应用性能等多个维度的监控和告警。

## 功能模块

### 1. 监控总览 (`/monitor/dashboard`)
- **系统资源概览**: CPU、内存、磁盘、网络使用情况
- **性能趋势图表**: 系统资源和网络性能趋势
- **服务状态**: 关键服务运行状态监控
- **数据库连接**: 实时数据库连接状态
- **最近告警**: 显示最近的告警信息

### 2. 系统监控 (`/monitor/system`)
- **实时指标**: CPU、内存、磁盘、网络使用率
- **详细图表**: CPU和内存使用趋势，支持1小时/6小时/24小时时间范围
- **进程监控**: TOP进程列表，按CPU使用率排序
- **服务管理**: 系统服务状态监控和控制
- **实时网络流量**: Canvas绘制的实时网络流量图

### 3. 网络监控 (`/monitor/network`)
- **响应时间**: 平均响应时间监控和趋势分析
- **吞吐量**: 请求处理能力统计
- **错误率**: 网络请求错误率监控
- **API性能**: 最慢和最热门API接口排行
- **状态码分析**: HTTP状态码分布统计

### 4. 数据库监控 (`/monitor/database`)
- **连接管理**: 活跃连接数、空闲连接数监控
- **查询性能**: 查询数量、响应时间统计
- **缓存监控**: 缓存命中率和使用情况
- **慢查询**: 慢查询SQL语句监控
- **锁监控**: 数据库锁等待和死锁监控
- **表空间**: 表空间使用情况分析

### 5. 应用监控 (`/monitor/application`)
- **用户活跃度**: 在线用户数、总用户数统计
- **页面性能**: 页面加载时间监控
- **业务指标**: 订单、销售额、用户增长等业务数据
- **异常监控**: 应用异常和错误统计
- **用户行为**: 会话时长、跳出率、转化率分析

### 6. 告警配置 (`/monitor/alert`)
- **告警规则**: 创建和管理性能监控告警规则
- **阈值设置**: 支持CPU、内存、响应时间等多种指标的阈值配置
- **通知方式**: 支持邮件、短信、钉钉、微信等多种通知方式
- **告警历史**: 告警记录查看和状态管理
- **规则管理**: 告警规则的启用/禁用、编辑、删除

## 技术特性

### 数据更新机制
- **自动刷新**: 所有监控页面支持自动数据刷新（5-30秒间隔）
- **实时图表**: 使用Canvas绘制实时更新的图表
- **数据缓存**: 合理的数据缓存机制，减少服务器压力

### 可视化组件
- **ECharts集成**: 使用v-charts组件库，支持多种图表类型
- **响应式设计**: 适配不同屏幕尺寸，支持移动端访问
- **交互式图表**: 支持时间范围选择、图表缩放等交互功能

### 告警系统
- **多维度监控**: 支持系统、网络、数据库、应用等多个维度
- **灵活配置**: 可自定义监控指标、阈值和通知方式
- **状态管理**: 告警状态跟踪和历史记录

## 使用方法

### 启动系统
```bash
npm install
npm run dev
```

### 访问监控
- 登录系统后，在左侧菜单找到"性能监控"模块
- 点击对应的子菜单进入不同的监控页面
- 监控总览页面提供全局性能状态概览

### 配置告警
1. 进入"告警配置"页面
2. 点击"添加规则"按钮
3. 选择监控指标、设置阈值和通知方式
4. 保存规则并启用

### 查看历史数据
- 大部分图表支持时间范围选择
- 可以查看1小时、6小时、24小时的历史趋势
- 告警历史提供详细的告警记录

## API接口

### 数据接口
所有监控数据通过`/src/api/monitor.js`中定义的接口获取：

- `getSystemMetrics()`: 系统资源指标
- `getNetworkMetrics()`: 网络性能指标  
- `getDatabaseMetrics()`: 数据库性能指标
- `getApplicationMetrics()`: 应用性能指标
- `getAlertConfig()`: 告警配置
- `getAlertHistory()`: 告警历史

### 模拟数据
当前使用模拟数据进行演示，实际使用时需要替换为真实的后端API接口。

## 扩展说明

### 添加新的监控指标
1. 在`monitor.js`中添加对应的API接口
2. 在相应的Vue组件中调用新接口
3. 更新图表配置和数据处理逻辑

### 自定义图表
- 使用v-charts的配置项自定义图表样式
- 可以添加新的图表类型（柱状图、饼图、散点图等）
- 支持图表的动画效果和交互功能

### 集成真实后端
- 替换模拟数据为真实API调用
- 添加错误处理和loading状态
- 实现数据缓存和优化策略

## 注意事项

1. **性能考虑**: 避免过于频繁的数据刷新，建议5-30秒间隔
2. **内存管理**: 及时清理定时器，避免内存泄漏
3. **错误处理**: 添加适当的错误处理机制
4. **权限控制**: 根据需要添加权限验证
5. **数据安全**: 敏感数据需要加密传输
